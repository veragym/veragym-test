Phase 4

회원 사진 기능 수정

iOS 회원 앱에서 사진 불러오기 실패 — 3번 반복 수정 후 최종 해결

  • 문제 — 회원 앱(iOS)에서 트레이너가 찍은 사진 불러오기 실패
  • 1차 시도 실패 — split_part(m.name, '/', 2)
    members.name 컬럼을 경로로 착각
  • 2차 시도 실패 — split_part(name, '/', 2)
    EXISTS 서브쿼리 내에서 PostgreSQL이 name을 members.name으로 resolve
  • 3차 시도 실패 — IN 방식
    members 테이블에 RLS 정책으로 anon이 SELECT 불가
  • 최종 해결: SECURITY DEFINER 함수
    CREATE OR REPLACE FUNCTION storage_is_active_member(p_member_id TEXT)
    RETURNS BOOLEAN LANGUAGE SQL SECURITY DEFINER SET search_path = public
    AS $$ SELECT EXISTS (SELECT 1 FROM members WHERE id::text = p_member_id AND is_active = true); $$;
    • postgres 권한으로 실행되어 RLS 우회 가능
    • 이후 추가되는 트레이너/회원 모두 자동 적용
  • 사진 로딩 속도 개선 — createSignedUrls 배치 API로 1회 호출로 다수 URL 서명

주요 기술적 결정사항

🔄
bfcache 대응 패턴
_initRunning 플래그 + pageshow 리셋 패턴을 모든 페이지에 적용하여 iOS/PWA 백그라운드 캐시 문제 해결

Network-First Service Worker
HTML/JS는 항상 최신 버전 로드 (network-first), 이미지·아이콘·매니페스트는 cache-first로 성능 최적화
🔐
SECURITY DEFINER 함수
anon 사용자가 RLS 보호된 테이블 참조 필요 시 postgres 권한으로 실행되는 함수로 안전하게 우회
🎯
단일 진입점 원칙
기능 진입 경로는 UX 맥락에 맞는 1개로 단순화하여 사용자 혼동 제거
📊
배치 API 활용
개별 요청 대신 배치 API (createSignedUrls)로 다수 리소스 서명, 로딩 속도 크게 개선

3개 앱(트레이너·회원·관리자) 간 일관된 UI 패턴 적용으로 사용자 학습곡선 감소
